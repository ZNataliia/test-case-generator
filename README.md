# Test Case Generator

A small Streamlit tool that turns a pasted requirement or user story into a structured set of test cases, generated by Claude via the Anthropic API.

## What it does

1. You paste a requirement or user story into a text box.
2. Click **Generate Test Cases**.
3. The app sends the requirement to Claude (`claude-sonnet-5`) using a prompt template (`prompts/template.md`) and asks for a structured JSON response (positive, negative, and edge cases).
4. Results are shown as a table (ID, Title, Type, Preconditions, Steps, Expected Result, Priority) and can be downloaded as a CSV.
5. If the requirement is ambiguous, Claude flags what's unclear instead of guessing, and those notes are shown above the table.

## Project structure

```
test-case-generator/
  app.py              # Streamlit UI + logic
  prompts/
    template.md        # prompt template sent to Claude
  tests/
    test_app.py        # pytest suite (no API key/network needed)
  .env                # holds ANTHROPIC_API_KEY (gitignored)
  .gitignore
  requirements.txt
  README.md
```

## Setup

1. **Install dependencies** (ideally in a virtual environment):

   ```bash
   cd test-case-generator
   python -m venv venv
   venv\Scripts\activate        # Windows
   # source venv/bin/activate   # macOS/Linux
   pip install -r requirements.txt
   ```

2. **Add your API key.** Open `.env` and replace the placeholder:

   ```
   ANTHROPIC_API_KEY=sk-ant-...
   ```

   Get a key from the [Anthropic Console](https://console.anthropic.com/). `.env` is gitignored, so your key won't be committed.

3. **Run the app:**

   ```bash
   streamlit run app.py
   ```

   Streamlit will open the app in your browser (usually `http://localhost:8501`).

## Running tests

```bash
pytest
```

The suite doesn't call the real Anthropic API — it uses a fake stream object to test the cancellation logic and validates the JSON schema directly, so it runs fast and doesn't need an API key.

## Notes

- The app uses the API's [structured outputs](https://platform.claude.com/docs/en/build-with-claude/structured-outputs) feature (`output_config.format`) so Claude's response is guaranteed to match a fixed JSON schema — this is what makes parsing into a table reliable instead of hoping Claude's prose happens to be valid JSON.
- If you see an authentication error, double check the key in `.env` and restart the Streamlit app (it only reads `.env` on startup).
